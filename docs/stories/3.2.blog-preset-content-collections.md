# Story 3.2: Blog Preset (Content Collections)

## Status
Done

## Story
**As a** solo freelancer,  
**I want** a blog preset with content collections,  
**so that** I can publish posts quickly.

## Acceptance Criteria
1. Blog preset includes a posts content collection schema using Astro Content Collections.
2. Blog output includes list, detail, tag/category, and RSS routes.
3. Static build succeeds with no backend dependencies.

## Tasks / Subtasks
- [ ] Define blog preset composition (AC: 1, 2, 3)
  - [ ] Create preset metadata in `templates/presets/blog/`.
  - [ ] Use a YAML preset manifest named `preset.yaml`.
  - [ ] Reference base template and add blog pages/content only in preset scope.
- [ ] Add content collection schema (AC: 1)
  - [ ] Add `templates/presets/blog/src/content/config.ts` defining the `posts` collection.
  - [ ] Include required fields: `title`, `description`, `pubDate`, `tags` (array), `category`.
  - [ ] Add sample post content under `templates/presets/blog/src/content/posts/`.
- [ ] Add blog routes (AC: 2)
  - [ ] List page: `/blog` -> `templates/presets/blog/src/pages/blog/index.astro`
  - [ ] Detail page: `/blog/[slug]` -> `templates/presets/blog/src/pages/blog/[slug].astro`
  - [ ] Tag page: `/blog/tags/[tag]` -> `templates/presets/blog/src/pages/blog/tags/[tag].astro`
  - [ ] Category page: `/blog/categories/[category]` -> `templates/presets/blog/src/pages/blog/categories/[category].astro`
  - [ ] RSS: `/rss.xml` -> `templates/presets/blog/src/pages/rss.xml.ts`
  - [ ] Use the base layout (`templates/base/src/layouts/BaseLayout.astro`) in all blog pages.
- [ ] Verify static-only output (AC: 3)
  - [ ] Ensure no backend deps or frameworks are introduced.
  - [ ] Run `pnpm build` for static output (base template).
