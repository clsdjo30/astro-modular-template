# Story 5.2: o2switch Deployment Preset (Static UI + API, Optional SSR)

## Status
Done

## Story
**As a** solo freelancer,  
**I want** an o2switch deployment preset,  
**so that** I can host static UI plus a self-hosted JSON API on my infrastructure.

## Acceptance Criteria
1. Preset documents static UI deployment plus separate Node.js API service on o2switch.
2. API is the default for backend-enabled presets; Astro SSR/hybrid is optional.
3. Preset includes explicit Node.js version, start command, and env var configuration notes for o2switch.

## Tasks / Subtasks
- [ ] Define o2switch preset scope (AC: 1, 2)
  - [ ] Create preset metadata in `templates/presets/o2switch/`.
  - [ ] Document default model: static Astro UI + separate JSON API service.
  - [ ] Add optional note for SSR/hybrid only when explicitly enabled.
- [ ] Add deployment guidance (AC: 1, 3)
  - [ ] Document Node.js LTS version and `pnpm start` for API service.
  - [ ] Document env var configuration for API and UI.
  - [ ] Note build output paths for UI (`dist/`) and API (compiled `dist/` or `build/`).
- [ ] Validate compatibility constraints (AC: 2)
  - [ ] Ensure preset does not imply Astro SSR by default.
